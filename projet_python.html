Analyse Logs Web & BDD

Conception et mise en place d'une solution complète pour le traitement, l'intégration et l'analyse de journaux d'accès web (logs). Ce projet allie le développement de scripts d'automatisation en Python et l'administration système/réseaux sous environnement Linux.

Python SQL MariaDB Rocky Linux 9 Apache phpMyAdmin

Le Projet

Contexte

Les serveurs web (comme Apache) génèrent quotidiennement d'énormes fichiers textes appelés "logs", contenant toutes les requêtes des utilisateurs. L'objectif de ce projet était de créer des outils pour parser ces fichiers complexes et les transformer en données statistiques intégrables dans une base de données sécurisée.

Objectifs techniques

  • Configurer une Machine Virtuelle (VM) Linux et l'intégrer au réseau.
  • Déployer un serveur de base de données MariaDB.
  • Développer des scripts Python capables de lire, découper et analyser de lourds fichiers textes.
  • Générer dynamiquement des fichiers `.sql` d'insertion de données.

Sécurisation & Administration

  • Application du principe du "moindre privilège" en créant des utilisateurs SQL distincts (un accès en lecture seule `utilog`, un accès complet `gestionlog`).
  • Configuration du serveur Apache et de phpMyAdmin pour n'autoriser les connexions distantes que depuis une adresse IP spécifique.

Traitement de données (Python)

  • Générateur de pseudos : Script lisant un fichier texte pour générer automatiquement des pseudonymes basés sur le nom, prénom et âge.
  • Analyseur de requêtes : Script avec menu interactif identifiant les URLs les plus visitées pour chaque adresse IP.
  • Convertisseur : Transformation des lignes de logs bruts en requêtes `INSERT INTO` SQL.
