Android Mobile API REST

Application Android R3st0

Application mobile Android de réservation et consultation de restaurants développée en équipe avec architecture MVC, API REST PHP et gestion GitLab en méthode agile.

Java Android Android Studio API REST PHP JSON MariaDB GitLab Canva

Description & Contexte

Mise en situation et objectifs du projet

Contexte du projet

Cette application Android mobile est le prolongement du site web R3st0.fr. Elle permet aux utilisateurs de consulter la liste des restaurants, visualiser leurs fiches détaillées (photos, avis, notes, horaires, adresse) et préparer de futures réservations directement depuis leur smartphone.

Objectifs du projet

  • Développer une application Android native en Java avec architecture MVC
  • Créer une API REST en PHP pour exposer les données du site web aux applications mobiles
  • Concevoir des maquettes graphiques cohérentes avec l'identité visuelle de R3st0.fr
  • Implémenter la consommation d'API avec requêtes HTTP asynchrones (AsyncTask)
  • Travailler en équipe avec méthodologie agile (sprints, daily scrums, tickets GitLab)

Fonctionnalités développées

  • Liste des restaurants : Affichage scrollable de tous les restaurants avec photo, nom et ville
  • Fiche détaillée : Vue complète d'un restaurant (description, adresse, horaires, types de cuisine, galerie photos)
  • Avis et notes : Affichage des critiques utilisateurs avec système d'étoiles et calcul de note moyenne
  • Navigation intuitive : Retour arrière, transitions entre écrans, gestion des erreurs réseau

Architecture technique

L'application suit une architecture MVC (Modèle-Vue-Contrôleur) :

  • Modèle : Classes métier (Restaurant, Critique) et DAO pour la récupération des données depuis l'API
  • Vue : Layouts XML (activity_main.xml, activity_detail.xml) avec ScrollView, LinearLayout, ImageView
  • Contrôleur : Activités Java (MainActivity, RestoDetailActivity) gérant la logique applicative

API REST PHP

Une API REST a été développée en PHP pour exposer les données de la base MariaDB :

  • getAllRestos.php : Retourne la liste complète des restaurants en JSON
  • getOneById.php : Retourne les détails d'un restaurant (infos, photos, types de cuisine, critiques)
  • getNoteMoyenne.php : Calcule et retourne la note moyenne d'un restaurant
  • Gestion des erreurs HTTP (400, 404, 500) avec messages JSON explicites

Design et maquettage

  • Maquettes réalisées sur Canva en respectant la charte graphique de R3st0.fr (rouge/noir)
  • Interface épurée et moderne avec hiérarchie visuelle claire
  • Adaptation du design pour écrans mobiles (responsive, touch-friendly)

Technologies et outils

  • IDE : Android Studio pour le développement mobile
  • Langage : Java Android avec API niveau 21+ (Lollipop)
  • Backend : PHP 7+ pour l'API REST, MariaDB pour la base de données
  • Communication : HttpURLConnection, JSONObject, BufferedReader
  • Gestion de version : GitLab avec branches par itération
  • Maquettage : Canva pour les prototypes graphiques

Itérations réalisées

  • Itération 1 : Création du diagramme UML (architecture MVC), développement de l'API REST, maquettes Canva
  • Itération 2 : Affichage de la fiche détaillée d'un restaurant, connexion avec l'API, gestion des critiques et notes

Organisation en équipe

Travail en trinôme (Nikauly, Garance, Heymad) avec daily scrums hebdomadaires :

  • Répartition des tâches via tickets GitLab (Issues Board)
  • Nikauly : API REST PHP, connexion Android-API, formatage des données
  • Garance : Endpoints API (getOneById, getNoteMoyenne), requêtes SQL
  • Heymad : Layouts XML, design des vues, maquettes Canva
