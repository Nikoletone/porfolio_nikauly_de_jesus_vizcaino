THALI-MINI

Application Java de bureau pour la gestion des mini-excursions d'un centre de thalassothérapie avec architecture DAO et base de données MariaDB.

Java NetBeans MariaDB / MySQL Pattern DAO Swing / JFrame CRUD

Le Projet

Contexte

THALI est un centre de thalassothérapie souhaitant proposer un catalogue de mini-excursions organisées l'après-midi pour ses curistes. En tant que stagiaire chez STESIO (ESN mandatée), j'ai développé le module THALI-MINI pour gérer ces excursions et leurs étapes.

Objectifs du projet

  • Développer une application Java orientée objet avec architecture en couches.
  • Implémenter un CRUD complet sur les excursions et leurs étapes.
  • Utiliser le pattern DAO pour l'accès à la base de données MariaDB.
  • Créer une interface graphique utilisateur avec Swing (JFrame, JDialog).
  • Gérer les relations entre entités (une excursion contient plusieurs étapes ordonnées).

Fonctionnalités implémentées

  • Consultation de la liste des excursions avec leurs informations détaillées.
  • Ajout, modification et suppression d'excursions.
  • Gestion complète des étapes d'une excursion (CRUD).
  • Réorganisation de l'ordre des étapes (déplacement haut/bas).
  • Calcul automatique de la durée totale d'une excursion.
  • Validation des données saisies avant enregistrement en base.

Architecture technique

  • Couche GUI (paquetage "gui") : interfaces graphiques JFrame et JDialog pour l'interaction utilisateur.
  • Couche Métier (paquetage "modele.metier") : classes MiniExcursion et Etape représentant les données.
  • Couche DAO (paquetage "modele.dao") : classes d'accès aux données avec requêtes SQL.
